- 웹 소켓은 단일 TCP 연결을 통해 클라이언트와 서버 사이에 전이중 방향 통신을 할수있게 한다.

- HTTP 환경에서 작동하며 Spring 스펙에서 등장한 네트워크 서비스다.

- 기존 채팅 구현시 일반적인 자바소켓을 사용하여 소켓 통신 과정을 일일이 구현해야 했다.

- 웹 소켓은 연결요청에 대해 HTTP를 통해 Switching, HandShaking이 이루어진다.

- TCP는 이진 데이터만 주고받을수 있으나 웹소켓은 Binary와 Text 데이터도 주고 받을 수 있다.

- 웹 소켓에 큰 특징
. HTTP 통신의 단점 개선
. 영구적 양방향 통신
. HTML 5의 주요 API
. HTTP 프로토콜을 기반으로 하는 웹 브라우저의 웹서버간 양방향 통신을 지원하기 위한 표준
. Client/Server가 실시간으로 데이터를 주고 받을 수 있다.

아파치(apache)란?

- 웹 서버중 하나이며, 아파치 소프트웨어 재단에서 관리하는 HTTP 웹 서버이다.

톰캣(Tomcat)이란?

- 웹 어플리케이션 서버(와스)로서, 자바 서블릿을 실행시키고 JSP코드가 포함되어 있는 웹 페이지를 만들어준다.

톰캣은 웹 서버에서 넘어온 동적인 페이지를 읽어서 프로그램을 실행시키고, 그 결과를 다시 html로 재구성하여 아파치에게
되돌려 준다.

톰캣은 자체적으로 보유하고 있는 내부 웹 서버와 함께 독립적으로 사용될 수도 있지만, 아파치나 넷스케이프 엔터프라이즈 서버, 
IIS등 다른 웹 서버와 함께 사용될 수도 있다. 톰캣을 실행시키기 위해서는 JRE 1.1이상에 부합되는 자바 런타임 환경이 필요함.

아파치(웹서버) 와 톰캣(웹어플리케이션 서버)의 차이점

아파치는 말그대로 웹서버이고 톰캣은 웹서버와 서블릿컨테이너를 합친 웹어플리케이션서버이다.

즉, 웹서버는 정적인 데이터를 처리하는 서버이고,

웹 어플리케이션 서버는 동적인 데이터를 처리하는 서버이다.
그래서 톰캣을 실행하여 가장 먼저 읽어들이는 내용은 Web.xml 파일이다.

1. Websocket 만든 후 구조 파악 

2. spring pom.xml, servlet-context.xml, root-context.xml, web-xml 4개 파일 구조와 구성 파악
