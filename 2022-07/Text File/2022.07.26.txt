deploy - 배치하다.

web.xml은 JSP나 Servlet을 위한 설정파일이며 URL 경로와 해당 경로의 요청을 처리하는
서블릿 사이의 매핑을 정의한다. 웹 서버는 이 구성을 사용하여 특정한 요청을 처리할
서블릿을 식별하고 요청 메서드에 해당하는 클래스 메서드를 호출한다.

동적 바인딩 

- 다형성을 사용해서 메소드를 호출할 때, 발생하는 현상
- 실행 시간(런타임) 즉, 파일을 실행하는 시점에 성격이 결정된다.
- 실제 참조하는 객체는 서브 클래스이니 서브 클래스의 메소드를 호출한다.

정적 바인딩

- 컴파일 시간에 성격이 결정된다.
- 변수의 타입이 수퍼 클래스이니 수퍼 클래스의 메소드를 호출한다.

그렇다면 바인딩이란 무엇인가?

컴퓨터 프로그래밍에서 각종 값들이 확정되어 더 이상 변경할 수 없는 구속(bind) 상태가 되는것.
프로그램 내에서 식별자가 그 대상인 메모리 주소, 데이터형 또는 실제값으로 배정되는 것이다.

즉 동적 바인딩은 프로그램의 실행되는 과정에서 바인딩되는 것을 의미하며

즉 정적(static) 바인딩은 프로그램의 컴파일 또는 링크 시에 확정되는 바인딩이 되는것을 의미한다.
 
할당과 바인딩의 차이

메모리 할당 = 메모리를 확보하는것
바인딩 = 변수에 변수와 관련된 속성을 연관 시키는 과정.

정적 메모리 할당 = stack영역에 메모리를 확보하는 것. ex) int data
정적 바인딩 = 해당 변수에 변수와 관련된 값을 저장하는 것 ex int data = 5;

동적 메모리 할당 = heap 영역에 메모리를 확보하는 것. ex) new Object()
동적 바인딩 = 참조변수 또는 포인터로 힙영역의 객체를 연결시켜주는 것 ex) Object ob = new Obkect()

TCP/IP = 인터넷과 이와 유사한 컴퓨터 네트워크 사이에서 정보를 주고받는 데 이용되는
통신 프로토콜의 모음이다.
.TCP는 전송 제어 프로토콜 
.IP 는 인터넷 프로토콜 이다.

HTTP 프로토콜 = 통신규약

네트워크 = 몇 개의 독립적인 장치가 적절한 영역내에서 작당히 빠른 속도의 물리적
통신 채널을 통하여 서로가 직접 통신할 수 있도록 지원해 주는 데이터 통신 체계

Generic(제네릭) = 제네릭 타입은 타입을 파라미터로 가지는 클래스와 인터페이스다.
(타입을 명시해준다.)

인터넷 = 컴퓨터 통신망

네트워크와 인터넷의 차이는 단위의 차이다. 인터넷 > 네트워크

네트워크는 서로통신할 수 있는 장치의 집합이다.
인터넷은 서로통신할 수 있는 네트워크의 집합이다.

WAS는 클라이언트와 서버간의 소켓 통신에 TCP/IP 연결 관리와 HTTP 프로토콜 해석 등의
네트워크 기반 작업을 추상화해 일종의 실행 환경을 제공한다. 따라서 개발자는 TCP/IP
연결을 직접 생성하고 HTTP 프로토콜을 해석하는 과정을 생략하여 웹을 쉽게 구현 할 수 있다.
즉 WAS가 제공하는 추상화된 API로 네트워크에 접근한다.

서블릿이란 원칙적으로 javax.servlet.Servlet 인터페이스를 구현한 것이 서블릿이다. 
서블릿은 서블릿 컨테이너 내에 등록된 후 서블릿 컨테이너에 의해 생성, 호출, 소멸이 
이뤄진다. 다시 말해서 서블릿은 독립적으로 실행되지 않고 main 메서드가 필요하지 않으며 서블릿
컨테이너에 의해 서블릿의 상태가 바뀐다.

Apache Tomcat 은 오픈소스 서블릿 컨테이너이다.
